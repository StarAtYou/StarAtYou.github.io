二、 获取文件

1、获取文件的 URL

  1. 同上，找到 IptvDeBug 类。

  2. 看打印标识。

  3. 找到标识的 InitActivity 类。

  4. 这个类里面有 onPageFinished 可以用来判断当前页面的 URL。

  5. 可以在串口里输入命令里快速定位 URL。

2、用 wireshark 解析抓到的包，获得文件

  1. 用 wireshark 打开刚刚抓到的包，输入 http 来过滤捕捉到的内容。

  2. 按 Ctrl + F，在搜索框里输入 URL（就是文件的路径），点击搜索框左边的下拉框：选择“字符串”或者“正则表达式”来过两次查找内容。

  3. 找到请求文件。

- 如果抓到了完整的文件

1 可以双击 Response。

2 会调到与之相对应的回复。

3 然后在下面这个区域右键，选择“导出字节流”即可。

- 如果抓不到文件

  1. 可以用追踪流来看是否有代码。
    如：我们想要看 /iptvepg/frame128/page/channelosd/blank.jsp 的代码。

  2. 可以右键选择“追踪流”，看看其中的 TCP 和 HTTP 有没有代码。

  3. 进去后查找刚刚的 URL：/iptvepg/frame128/page/channelosd/blank.jsp。

  4. 查找到后，下面蓝色的部分便是代码。

  5. 但是这样的代码有一定的乱码和缺失，一般不建议使用。

  1. log没有报错，根据问题描述来分析，判断可能是按返回键后的逻辑有问题，抓包-找到相关的文件，查看按键事件。

  3. 在相关地方加上打印后，发现了问题原因。

· addJavascriptInterface 解释：
需要在 JS 代码中访问 Java 中的函数，则会用到 WebView 的 addJavascriptInterface() 函数。
因为安全问题，在 Android4.2 中（如果应用的 android:targetSdkVersion 数值为 17+）JS只能访问带有 @JavascriptInterface 注解的 Java 函数。

· Handler 来处理发送过来的 msg，这里面就有播放器服务连接，播放器初始化，开始游戏等处理。
